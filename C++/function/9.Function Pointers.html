# Function Pointers

指针是一个变量，其中包含另一个变量的地址。函数指针相似，不同之处在于它们不是指向变量，而是指向函数！

​ 考虑以下函数：

int foo()
{
    return 5;
}

​ 标识符foo是函数的名称。但是函数是什么类型的？函数具有自己的左值函数类型-在这种情况下，该函数类型返回整数且不带参数。与变量非常相似，函数位于内存中的分配地址上。

​ 一个函数被调用时（通过()运算符），执行跳转到被调用函数的地址：

int foo() // code for foo starts at memory address 0x002717f0
{
    return 5;
}
 
int main()
{
    foo(); // jump to address 0x002717f0
 
    return 0;
}

​ 在您的编程生涯的某个时候（如果还没有），您可能会犯一个简单的错误：

#include <iostream>
 
int foo() // code starts at memory address 0x002717f0
{
    return 5;
}
 
int main()
{
    std::cout << foo << '\n'; // we meant to call foo(), but instead we're printing foo itself!
 
    return 0;
}

​ 我们无意中将函数foo直接发送到std :: cout，而不是调用函数foo()并打印返回值。在这种情况下会发生什么？

​ 在作者的机器上，则打印0x002717f0.

​ 但是它可能会在您的计算机上打印其他值（例如1），具体取决于编译器决定如何将函数指针转换为其他类型进行打印。如果您的机器没有打印该函数的地址，则可以通过将函数转换为空指针并打印以下内容来强制执行此操作：

#include <iostream>
 
int foo() // code starts at memory address 0x002717f0
{
    return 5;
}
 
int main()
{
    std::cout << reinterpret_cast<void*>(foo) << '\n'; // Tell C++ to interpret function foo as a void pointer
 
    return 0;
}

​ 就像可以声明指向普通变量的非恒定指针一样，也可以声明指向函数的非恒定指针。在本课程的其余部分中，我们将研究这些函数指针及其用法。函数指针是一个相当高级的主题，仅寻找C ++基础知识的人员可以安全地跳过或略读本课程的其余部分。

Pointers to functions

​ 创建非常量函数指针的语法是您在C ++中见过的最丑陋的事情之一：

// fcnPtr is a pointer to a function that takes no arguments and returns an integer
int (*fcnPtr)();

​ 在上面的代码段中，fcnPtr是指向没有参数且返回整数的函数的指针。 fcnPtr可以指向与此类型匹配的任何函数。

​ 出于优先原因，必须在* fcnPtr周围加上括号，因为int *fcnPtr()将被解释为名为fcnPtr的函数的前向声明，该函数不带参数并返回指向整数的指针。

​ 要创建const函数指针，const在星号之后：

int (*const fcnPtr)();

​ 如果将const放在int之前，则表明所指向的函数将返回const int。

Assigning a function to a function pointer

​ 可以使用函数初始化函数指针（并且可以为非const函数指针分配一个函数）。在上面的示例中，我们直接使用了foo，并将其转换为函数指针。与指向变量的指针一样，我们也可以使用＆foo获取指向foo的函数指针。为了与变量指针和成员函数指针保持一致（我们将在以后的课程中介绍），我们将使用＆foo语法，而不仅仅是foo。

int foo()
{
    return 5;
}
 
int goo()
{
    return 6;
}
 
int main()
{
    int (*fcnPtr)(){ &foo }; // fcnPtr points to function foo
    fcnPtr = &goo; // fcnPtr now points to function goo
 
    return 0;
}

​ 一个常见的错误就是：

fcnPtr = goo();

​ 实际上，这会将调用goo()函数的返回值分配给fcnPtr，这不是我们想要的。我们希望为fcnPtr分配函数goo的地址，而不是函数goo()的返回值。因此，不需要括号。

​ 请注意，函数指针的类型（参数和返回类型）必须与函数的类型匹配。以下是一些示例：

// function prototypes
int foo();
double goo();
int hoo(int x);
 
// function pointer assignments
int (*fcnPtr1)(){ &foo }; // okay
int (*fcnPtr2)(){ &goo }; // wrong -- return types don't match!
double (*fcnPtr4)(){ &goo }; // okay
fcnPtr1 = &hoo; // wrong -- fcnPtr1 has no parameters, but hoo() does
int (*fcnPtr3)(int){ &hoo }; // okay

​ 与基本类型不同，C ++会在需要时将函数隐式转换为函数指针（因此您无需使用地址运算符（＆）来获取函数的地址）。但是，它不会将函数指针隐式转换为void指针，反之亦然。

Calling a function using a function pointer

​ 您可以使用函数指针执行的另一项主要操作是使用它实际调用函数。有两种方法可以做到这一点。第一种是通过显式取消引用：

int foo(int x)
{
    return x;
}
 
int main()
{
    int (*fcnPtr)(int){ &foo }; // Initialize fcnPtr with function foo
    (*fcnPtr)(5); // call function foo(5) through fcnPtr.
 
    return 0;
}

​ 第二种方法是通过隐式解引用:

int foo(int x)
{
    return x;
}
 
int main()
{
    int (*fcnPtr)(int){ &foo }; // Initialize fcnPtr with function foo
    fcnPtr(5); // call function foo(5) through fcnPtr.
 
    return 0;
}

​ 如您所见，隐式解引用方法看起来就像一个普通的函数调用——这正是您所期望的，因为普通的函数名是指向函数的指针! 然而，一些较老的编译器不支持隐式解引用方法，但所有现代编译器都应该支持。

​ 一个有趣的注意:通过函数指针调用的函数并不能使用默认参数。默认形参在编译时解析(也就是说，如果不为默认形参提供实参，编译器在编译代码时为您替换一个实参)。然而，函数指针是在运行时解析的。因此，当使用函数指针调用函数时，无法解析默认形参。在这种情况下，必须显式地传入任何默认参数的值。

Passing functions as arguments to other functions

​ 用函数指针做的最有用的事情之一是将一个函数作为参数传递给另一个函数。作为另一个函数的参数的函数有时被称为回调函数(callback function)。

​ 考虑这样一种情况，您正在编写一个函数来执行一个任务(例如对数组进行排序)，但您希望用户能够定义该任务的特定部分将如何执行(例如数组是按升序排序还是按降序排序)。让我们仔细研究一下这个问题，作为一个可以推广到其他类似问题的例子，它特别适用于排序。

​ 许多基于比较的排序算法都基于类似的概念:排序算法遍历一个数字列表，对数字对进行比较，并根据这些比较的结果对数字重新排序。因此，通过改变比较，我们可以改变算法排序的方式而不影响其余的排序代码。

#include <utility> // for std::swap
 
void SelectionSort(int *array, int size)
{
    // Step through each element of the array
    for (int startIndex{ 0 }; startIndex < (size - 1); ++startIndex)
    {
        // smallestIndex is the index of the smallest element we've encountered so far.
        int smallestIndex{ startIndex };
 
        // Look for smallest element remaining in the array (starting at startIndex+1)
        for (int currentIndex{ startIndex + 1 }; currentIndex < size; ++currentIndex)
        {
            // If the current element is smaller than our previously found smallest
            if (array[smallestIndex] > array[currentIndex]) // COMPARISON DONE HERE
            {
                // This is the new smallest number for this iteration
                smallestIndex = currentIndex;
            }
        }
 
        // Swap our start element with our smallest element
        std::swap(array[startIndex], array[smallestIndex]);
    }
}

​ 现在，为了让调用者决定如何进行排序，我们将不使用我们自己硬编码的比较函数，而是允许调用者提供自己的排序函数!这是通过函数指针完成的。

​ 因为调用者的比较函数将比较两个整数并返回一个布尔值，指向这样一个函数的指针应该是这样的:

bool (*comparisonFunc)(int , int);

​ 因此，我们将允许调用者将指向其所需比较函数的指针作为第三个参数传递给排序例程，然后我们将使用调用者的函数来进行比较。

 
// Note our user-defined comparison is the third parameter
void selectionSort(int *array, int size, bool (*comparisonFcn)(int, int))
{
    // Step through each element of the array
    for (int startIndex{ 0 }; startIndex < (size - 1); ++startIndex)
    {
        // bestIndex is the index of the smallest/largest element we've encountered so far.
        int bestIndex{ startIndex };
 
        // Look for smallest/largest element remaining in the array (starting at startIndex+1)
        for (int currentIndex{ startIndex + 1 }; currentIndex < size; ++currentIndex)
        {
            // If the current element is smaller/larger than our previously found smallest
            if (comparisonFcn(array[bestIndex], array[currentIndex])) // COMPARISON DONE HERE
            {
                // This is the new smallest/largest number for this iteration
                bestIndex = currentIndex;
            }
        }
 
        // Swap our start element with our smallest/largest element
        std::swap(array[startIndex], array[bestIndex]);
    }
}
 
// Here is a comparison function that sorts in ascending order
// (Note: it's exactly the same as the previous ascending() function)
bool ascending(int x, int y)
{
    return x > y; // swap if the first element is greater than the second
}
 
// Here is a comparison function that sorts in descending order
bool descending(int x, int y)
{
    return x < y; // swap if the second element is greater than the first
}
 
// This function prints out the values in the array
void printArray(int *array, int size)
{
    for (int index{ 0 }; index < size; ++index)
    {
        std::cout << array[index] << ' ';
    }
    
    std::cout << '\n';
}
 
int main()
{
    int array[9]{ 3, 7, 9, 5, 6, 1, 8, 2, 4 };
 
    // Sort the array in descending order using the descending() function
    selectionSort(array, 9, descending);
    printArray(array, 9);
 
    // Sort the array in ascending order using the ascending() function
    selectionSort(array, 9, ascending);
    printArray(array, 9);
 
    return 0;
}

​ 这很酷吧?我们已经让调用者能够控制选择排序的工作方式。调用者甚至可以定义自己的“奇怪的”比较函数:

bool evensFirst(int x, int y)
{
	// if x is even and y is odd, x goes first (no swap needed)
	if ((x % 2 == 0) && !(y % 2 == 0))
		return false;
 
	// if x is odd and y is even, y goes first (swap needed)
	if (!(x % 2 == 0) && (y % 2 == 0))
		return true;
 
        // otherwise sort in ascending order
	return ascending(x, y);
}
 
int main()
{
    int array[9]{ 3, 7, 9, 5, 6, 1, 8, 2, 4 };
 
    selectionSort(array, 9, evensFirst); //2 4 6 8 1 3 5 7 9
    printArray(array, 9);
 
    return 0;
}

​ 如您所见，在这种情况下使用函数指针提供了一种很好的方式，允许调用者将自己的功能“挂钩”到您以前编写和测试过的东西中，这有助于促进代码重用!以前，如果希望按降序排序一个数组，按升序排序另一个数组，则需要多个版本的排序例程。现在，您可以有一个版本，可以按照调用者希望的方式进行排序!

​ 注意:如果函数形参属于函数类型，它将被转换为指向该函数类型的指针。这意味着

void selectionSort(int *array, int size, bool (*comparisonFcn)(int, int))

​ 可以等价地写成:

void selectionSort(int *array, int size, bool comparisonFcn(int, int))

​ 这只适用于函数参数，而不是独立的函数指针，因此用途有限。

Providing default functions

​ 如果您打算允许调用者将函数作为参数传入，那么为调用者提供一些方便使用的标准函数通常是很有用的。例如，在上面的选择排序示例中，提供升序()和降序()函数以及selectionSort()函数将使调用者的工作更简单，因为他们不必每次想使用它们时都重写升序()或降序()。

​ 你甚至可以将其中一个设置为默认参数:

// Default the sort to ascending sort
void selectionSort(int *array, int size, bool (*comparisonFcn)(int, int) = ascending);

​ 在这种情况下，只要用户正常调用selectionSort(不是通过函数指针)，comparisonFcn参数默认为ascending。

Making function pointers prettier with type aliases

​ 让我们面对它——指向函数的指针的语法很丑陋。然而，类型别名可以使函数指针看起来更像常规变量:

using ValidateFunction = bool(*)(int, int);

​ 这定义了一个名为ValidateFunction的类型别名，它是一个指向函数的指针，该函数接受两个int类型并返回bool类型。

bool validate(int x, int y, bool (*fcnPtr)(int, int)); // ugly
bool validate(int x, int y, ValidateFunction pfcn) // clean

**Using std::function **

​ 定义和存储函数指针的另一种方法是使用std::function，它是标准库<functional>头文件的一部分。要使用此方法定义函数指针，请声明一个std::function对象，如下所示:

#include <functional>
bool validate(int x, int y, std::function<bool(int, int)> fcn); // std::function method that returns a bool and takes two int parameters

​ 如您所见，返回类型和参数都在尖括号内，参数则在圆括号内。如果没有参数，括号可以为空。虽然这读起来有点冗长，但它也更清晰，因为它清楚地说明了预期的返回类型和参数是什么(而typedef方法使它们变得模糊)。

​ 用std::函数更新我们前面的例子:

#include <functional>
#include <iostream>
 
int foo()
{
    return 5;
}
 
int goo()
{
    return 6;
}
 
int main()
{
    std::function<int()> fcnPtr{ &foo }; // declare function pointer that returns an int and takes no parameters
    fcnPtr = &goo; // fcnPtr now points to function goo
    std::cout << fcnPtr() << '\n'; // call the function just like normal
 
    return 0;
}

​ 注意，你也可以输入别名std::function:

using ValidateFunctionRaw = bool(*)(int, int); // type alias to raw function pointer
using ValidateFunction = std::function<bool(int, int)>; // type alias to std::function

Type inference for function pointers

​ 就像auto关键字可以用来推断普通变量的类型一样，auto关键字也可以推断函数指针的类型。

#include <iostream>
 
int foo(int x)
{
	return x;
}
 
int main()
{
	auto fcnPtr{ &foo };
	std::cout << fcnPtr(5) << '\n';
 
	return 0;
}

​ 这与您预期的完全一样，而且语法非常清晰。当然，缺点是关于函数的形参类型和返回类型的所有细节都是隐藏的，因此在调用函数或使用其返回值时很容易出错。

Conclusion

​ 函数指针主要用于在数组(或其他结构)中存储函数，或需要将函数传递给另一个函数时。因为声明函数指针的本机语法很难看而且容易出错，所以我们建议使用std::function。在函数指针类型只使用一次的地方(例如单个参数或返回值)，可以直接使用std::function。在多次使用函数指针类型的地方，std::函数的类型别名是更好的选择(防止重复使用)。
