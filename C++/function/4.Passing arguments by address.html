# Passing arguments by address

​ 还有一种将变量传递给函数的方法，那就是通过地址。通过传递实参的地址涉及到传递实参变量的地址，而不是参数变量本身。因为实参是地址，所以函数形参必须是指针类型。然后，函数可以对指针进行解引用，以访问或更改所指向的值。

#include <iostream>
 
void foo(int *ptr)
{
    *ptr = 6;
}
 
int main()
{
    int value{ 5 };
 
    std::cout << "value = " << value << '\n'; // 5
    foo(&value);
    std::cout << "value = " << value << '\n';  // 6
    return 0;
}

​ 如您所见，函数foo（）通过指针参数ptr更改了参数的值（变量值）。按地址传递通常与指针一起使用，指针通常用于指向内置数组的开头。例如，以下函数将打印数组中的所有值：

void printArray(int *array, int length)
{
    //sizeof(array) = 8, sizeof(array) = 4
    for (int index{ 0 }; index < length; ++index)
    {
        std::cout << array[index] << ' ';
    }
}
int main()
{
    int array[6]{ 6, 5, 4, 3, 2, 1 }; // remember, arrays decay into pointers
    printArray(array, 6); // so array evaluates to a pointer to the first element of the array here, no & needed
}
/*
注意，这里通过地址传入只能获得当前地址，不能通过以下方式计算出数组大小
length = sizeof(array) / sizeof(array[0])
*/

请记住，固定数组在传递给函数时会衰减为指针，因此我们必须将长度作为单独的参数传递。

​ 在取消引用地址之前，确保通过地址传递的参数不是空指针始终是一个好主意。取消引用空指针通常会导致程序崩溃。这是带有空指针检查的我们的printArray()函数：

void printArray(int *array, int length)
{
    // if user passed in a null pointer for array, bail out early!
    if (!array)
        return;
 
    for (int index{ 0 }; index < length; ++index)
        std::cout << array[index] << ' ';pp
}
 
int main()
{
    int array[6]{ 6, 5, 4, 3, 2, 1 };
    printArray(array, 6);
}

Passing by const address

​ 因为printArray()不会修改其任何参数，所以将数组参数设为const是一种很好的形式：

void printArray(const int *array, int length)
{
    // if user passed in a null pointer for array, bail out early!
    if (!array)
        return;
 
    for (int index{ 0 }; index < length; ++index)
        std::cout << array[index] << ' ';
}
 
int main()
{
    int array[6]{ 6, 5, 4, 3, 2, 1 };
    printArray(array, 6);
}

​ Addresses are actually passed by value

当您将指针传递给函数时，该指针的值（其指向的地址）将从参数复制到函数的参数。换句话说，它是通过价值传递的！如果更改功能参数的值，则仅更改副本。因此，原始指针参数将不会更改。 这是一个说明此情况的示例程序。

#include <iostream>
 
void setToNull(int *tempPtr)
{
    // we're making tempPtr point at something else, not changing the value that tempPtr points to.
    tempPtr = nullptr; // use 0 instead if not C++11
}
 
int main()
{ 
    // First we set ptr to the address of five, which means *ptr = 5
    int five{ 5 };
    int *ptr{ &five };
	
    // This will print 5
    std::cout << *ptr;
 
    // tempPtr will receive a copy of ptr
    setToNull(ptr);
 
    // ptr is still set to the address of five!
 
    // This will print 5
    if (ptr)
        std::cout << *ptr;
    else
        std::cout << " ptr is null";
 
    return 0;
}

tempPtr接收ptr持有的地址的副本。即使我们将tempPtr更改为其他内容（nullptr），这也不会更改ptr指向的值。因此，该程序将打印： 55。

​ 请注意，即使地址本身是通过值传递的，您仍然可以解引用该地址以更改参数的值。这是一个常见的混淆点，因此让我们澄清一下：

  • 当按地址传递自变量时，函数参数变量将从自变量接收地址的副本。此时，函数形和实参都指向相同的值。
  • 如果随后解引用函数参数以更改所指向的值，则将影响参数所指向的值，因为函数参数和参数都指向相同的值！
  • 如果为函数参数分配了其他地址，则不会影响参数，因为函数参数是副本，更改副本不会影响原始地址。更改函数参数的地址后，函数参数和参数将指向不同的值，因此取消引用参数并更改值将不再影响参数所指向的
#include <iostream>
 
void setToSix(int *tempPtr)
{
    *tempPtr = 6; // we're changing the value that tempPtr (and ptr) points to
}
 
int main()
{ 
    // First we set ptr to the address of five, which means *ptr = 5
    int five{ 5 };
    int *ptr{ &five };
	
    // This will print 5
    std::cout << *ptr;
 
    // tempPtr will receive a copy of ptr
    setToSix(ptr);
 
    // tempPtr changed the value being pointed to to 6, so ptr is now pointing to the value 6
 
    // This will print 6
    if (ptr)
        std::cout << *ptr;
    else
        std::cout << " ptr is null";
 
    return 0;
}

Passing addresses by reference

​ 下一个问题是：“如果我们要更改函数中自变量指向的地址，该怎么办？”事实证明，这非常容易。您可以简单地通过引用传递地址。引用指针的语法有点奇怪（并且很容易倒退）。但是，如果确实将其倒退，编译器将给您一个错误。

#include <iostream>
 
// tempPtr is now a reference to a pointer, so any changes made to tempPtr will change the argument as well!
void setToNull(int *&tempPtr)
{
    tempPtr = nullptr; // use 0 instead if not C++11
}
 
int main()
{ 
    // First we set ptr to the address of five, which means *ptr = 5
    int five{ 5 };
    int *ptr{ &five };
	
    // This will print 5
    std::cout << *ptr;
 
    // tempPtr is set as a reference to ptr
    setToNull(ptr);
 
    // ptr has now been changed to nullptr!
 
    if (ptr)
        std::cout << *ptr;
    else
        std::cout << " ptr is null";
 
    return 0;
}

There is only pass by value

​ 引用通常由编译器实现为指针。这意味着在后台，按引用传递实质上只是一个按地址传递（通过对引用的访问进行隐式解引用）。

​ 在上面，我们显示了按地址传递实际上就是按值传递地址！

​ 因此，我们可以得出结论，C ++确实通过值传递了一切！通过地址（和引用）传递的属性仅来自以下事实：我们可以解引用传递的地址以更改参数，而使用常规值参数则无法做到这一点！

Pass by address makes modifiable parameters explicit

int foo1(int x); // pass by value
int foo2(int &x); // pass by reference
int foo3(int *x); // pass by address
 
int i {};
 
foo1(i);  // can't modify i
foo2(i);  // can modify i
foo3(&i); // can modify i

​ 从对foo2()的调用中可以看出，该函数可以修改变量i，不是吗？

​ 因此，一些指南建议按地址传递所有可修改的参数，以便从现有函数调用中可以明显看出可以修改参数。 但是，这样做有其自身的缺点：调用者可能认为他们可以在不应该使用的时候传递nullptr，现在您必须严格检查空指针。 我们倾向于通过引用传递非可选的可修改参数的建议。更好的是，完全避免修改参数。

Pros and cons of pass by address

  • 优点
    • 通过地址传递允许函数更改参数的值，这有时很有用。否则，可以使用const来确保函数不会更改参数。 （但是，如果要使用非指针执行此操作，则应改用按引用传递）。
    • 由于未复制参数，因此即使与大型结构或类一起使用时，它也很快速。(注意，仅仅复制了参数的地址，并没有其它信息，所以也很快速)。
    • 我们可以通过out参数从一个函数返回多个值。
  • 缺点
    • 因为字面值(2,3,etc)（C风格的字符串文字除外）和表达式没有地址，所以指针参数必须是普通变量。
    • 必须检查所有值以查看它们是否为空。尝试取消引用空值将导致崩溃。很容易忘记这样做。
    • 因为取消引用指针比直接访问值要慢，所以访问由地址传递的参数要比访问按值传递的参数慢。

When to use pass by address:

  • 传递内置数组时（如果您可以接受它们会衰减为指针的事实）。
  • 从逻辑上讲，传递指针和nullptr是有效的参数。

When not to use pass by address:

  • 从逻辑上讲，传递指针和nullptr时不是有效的参数（使用按引用传递）。
  • 传递结构或类时（使用按引用传递）。
  • 传递基本类型时（使用按值传递）。

​ 如您所见，按地址传递和按引用传递具有几乎相同的优点和缺点。由于按引用传递通常比按地址传递更安全，因此在大多数情况下应首选按引用传递。

在适用的情况下，建议先通过引用传递，再通过地址传递。
