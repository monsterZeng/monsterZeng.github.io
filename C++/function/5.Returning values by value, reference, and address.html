Returning values by value, reference, and address

peiji

# Returning values by value, reference, and address

本节中，我们将考虑通过所有三种方法将值返回给调用方的问题。

​ 事实证明，按值，地址或引用将函数的值返回给调用者的方法几乎与将参数传递给函数的方法完全一样。每种方法都具有相同的优点和缺点。两者之间的主要区别只是数据流的方向相反。但是，还增加了一点复杂性-因为函数中的局部变量超出范围并在函数返回时被破坏，所以我们需要考虑此操作对每种返回类型的影响。

# Return by value

​ 按值返回是最简单，最安全的返回类型。当按值返回值时，该值的副本将返回给调用方。与按值传递一样，您可以按值返回字面量（例如5），变量（例如x）或表达式（例如x + 1），这使得按值返回非常灵活。

​ 按值返回的另一个优点是，您可以返回涉及在函数中声明的局部变量的变量（或表达式），而不必担心范围问题。由于变量是在函数返回之前求值的，并且值的副本已返回给调用者，所以当函数的变量超出函数范围的范围时，这没有问题。

int doubleValue(int x)
{
    int value{ x * 2 };
    return value; // A copy of value will be returned here
} // value goes out of scope here

​ 当返回在函数内部声明的变量或返回通过值传递的函数参数时，按值返回最合适。但是，像按值传递一样，对于结构和大型类，按值返回很慢。

When to use return by value:

  • 返回在函数内部声明的变量时
  • 返回按值传递的函数参数时

When not to use return by value:

  • 返回内置数组或指针时（使用按地址返回）
  • 返回大型结构或类时（使用按引用返回）

# Return by address

​ 按地址返回涉及将变量的地址返回给调用方。与按地址传递类似，按地址返回只能返回变量的地址，而不能返回文字或表达式（没有地址）。由于按地址返回只是将地址从函数复制到调用方，因此按地址返回速度很快。

​ 但是，按地址返回还有一个缺点，而按值返回没有这个缺点——如果您尝试返回函数本地变量的地址，则程序将表现出未定义的行为。考虑以下示例：

int* doubleValue(int x)
{
    int value{ x * 2 };
    return &value; // return value by address here
} // value destroyed here

​ 如您在此处看到的，值在返回给调用方之后就被销毁。最终结果是，调用者最终以未分配的内存（悬挂的指针）的地址结尾，如果使用该地址将引起问题。这是新程序员经常犯的错误。如果程序员尝试按地址返回局部变量，许多较新的编译器都会发出警告（不是错误）-但是，有很多方法可以欺骗编译器，使您在不生成警告的情况下进行非法操作，因此确保程序返回的指针将在函数返回后指向有效变量是程序员的重担。

​ 按地址返回通常用于将动态分配的内存返回给调用方

int* allocateArray(int size)
{
    return new int[size];
}
 
int main()
{
    int *array{ allocateArray(25) };
 
    // do stuff with array
 
    delete[] array;
    return 0;
}

​ 这个之所以可行，是因为动态分配的内存在声明它的块的末尾不会超出范围，因此当地址返回给调用者时，内存仍将存在。跟踪手动分配可能很困难。将分配和删除分成不同的功能，这使得了解谁负责删除资源或根本是否需要删除资源变得更加困难。应该使用智能指针（稍后介绍）和在其自身之后清理的类型，而不是手动分配。

When to use return by address:

  • 返回动态分配的内存时和您不能使用为您处理分配的类型。
  • 返回通过地址传递的函数实数时。

When not to use return by address:

  • 返回在函数内部声明的变量或按值传递的参数时（使用按值返回）
  • 返回通过引用传递的大型结构或类时（使用按引用返回）

# Return by reference

​ 跟返回地址一样,通过引用返回值必须是变量(你不应该返回一个字面量的引用或表达式解析为一个临时的值，这些将超出函数结束的范围,它最终会返回一个悬空引用)。当变量通过引用返回时，对该变量的引用被传递回调用者。然后调用者可以使用这个引用继续修改变量，这有时是很有用的。引用返回也很快速，这在返回结构和类时很有用。

​ 但是，就像按地址返回一样，您不应该返回局部变量的引用。考虑下面的例子:

int& doubleValue(int x)
{
    int value{ x * 2 };
    return value; // return a reference to value here
} // value is destroyed here

​ 在上面的程序中，函数返回时，程序返回了一个对将被销毁的值的引用。这意味着调用者接收到一个对garbage的引用。幸运的是，如果您试图这样做，编译器可能会给您一个警告或错误。

​ 返回引用通常用于在通过引用传递给函数的参数返回给调用者。在下面的例子中，我们(通过引用)返回通过引用传递给函数的数组元素:

#include <array>
#include <iostream>
 
// Returns a reference to the index element of array
int& getElement(std::array<int, 25> &array, int index)
{
    // we know that array[index] will not be destroyed when we return to the caller (since the caller passed in the array in the first place!)
    // so it's okay to return it by reference
    return array[index];
}
 
int main()
{
    std::array<int, 25> array;
 
    // Set the element of array with index 10 to the value 5
    getElement(array, 10) = 5;
	
    std::cout << array[10] << '\n';
 
    return 0;
}

​ 当调用getElement(array, 10)时，getElement()返回索引为10的数组元素的引用。main()然后使用这个引用将该元素的值赋给5。

​ 虽然这是一个人为设计的示例(因为您可以直接访问数组[10])，但一旦您了解了类，您将发现更多通过引用返回值的用法。

When to use return by reference:

  • 返回引用参数时。
  • 返回通过引用或地址传递给函数的对象的成员时。
  • 当返回一个不会在函数结束时被销毁的大型结构体或类时(例如通过引用传入的结构体)。

When not to use return by reference:

  • 当返回函数内部声明的变量或按值传递的参数时(使用按值返回)。
  • 返回内置数组或指针值时(使用按地址返回)

# Mixing return references and values

​ 虽然函数可以返回值或引用，但调用者可以将结果赋值给相应的变量或引用，也可以不赋值。让我们看看混合使用值类型和引用类型会发生什么。

int returnByValue()
{
    return 5;
}
 
int& returnByReference()
{
     static int x{ 5 }; // static ensures x isn't destroyed when the function ends
     return x;
}
 
int main()
{
    int giana{ returnByReference() }; // case A -- ok, treated as return by value
    int& ref{ returnByValue() }; // case B -- compile error since the value is an r-value, and an r-value can't bind to a non-const reference
    const int& cref{ returnByValue() }; // case C -- ok, the lifetime of the return value is extended to the lifetime of cref
 
    return 0;
}

​ 在情况A中，我们将一个引用返回值赋给一个非引用变量。因为giana不是一个引用，所以返回值被复制到giana中，就像returnByReference()已经通过value返回一样。

​ 在B种情况下，我们尝试用returnByValue()返回的返回值的副本初始化reference ref。但是，由于返回的值没有地址(它是r值)，这将导致编译错误。

​ 在C案例中，我们尝试使用由returnByValue() 返回的返回值的副本来初始化const引用cref。因为const引用可以绑定到r值，所以这里没有问题。通常，右值在创建它们的表达式的末尾过期——但是，当绑定到const引用时，右值的生存期(在本例中是函数的返回值)被扩展，以匹配引用的生存期(在本例中是cref)

# Lifetime extension doesn’t save dangling references

const int& returnByReference()
{
     return 5;
}
 
int main()
{
    const int &ref { returnByReference() }; // runtime error
}

​ 在上面的程序中，returnByReference()返回一个指向函数结束时将超出作用域的值的const引用。这通常是一个禁忌，因为它将导致悬空引用。但是，我们也知道，将一个值赋给const引用可以延长该值的生存期。那么这里哪个优先呢?5先超出范围，还是ref延长5的生命?

​ 答案是5首先超出作用域，然后将对5的引用复制回调用者，然后ref扩展当前悬空引用的生命周期。

​ 然而，下面的工作确实如预期的那样:

const int returnByValue()
{
     return 5;
}
 
int main()
{
    const int &ref { returnByValue() }; // ok, we're extending the lifetime of the copy passed back to main
}

在本例中，文字值5首先被复制回调用者(main)的范围内，然后ref扩展该副本的生存期。

# Returning multiple values

​ c++不包含将多个值传递回调用者的直接方法。虽然有时你可以重新构建你的代码，这样你可以分别传递每个数据项(例如，不是让一个函数返回两个值，而是让两个函数每个返回一个值)，这可能是麻烦和不直观的。

​ 正如第3章(通过引用传递参数)中所介绍的，out参数提供了一个将多位数据传递回调用者的方法。我们不推荐这种方法。

​ 第二种方法涉及到使用纯数据结构体:

#include <iostream>
 
struct S
{
	int m_x;
	double m_y;
};
 
S returnStruct()
{
	S s;
	s.m_x = 5;
	s.m_y = 6.7;
	return s;
}
 
int main()
{
	S s{ returnStruct() };
	std::cout << s.m_x << ' ' << s.m_y << '\n';
 
	return 0;
}

​ 第三种方法(在c++ 11中引入)是使用std::tuple。元组是可能是不同类型的元素序列，其中每个元素的类型都必须显式指定。

#include <tuple>
#include <iostream>
 
std::tuple<int, double> returnTuple() // return a tuple that contains an int and a double
{
	return { 5, 6.7 };
}
 
int main()
{
	std::tuple s{ returnTuple() }; // get our tuple
	std::cout << std::get<0>(s) << ' ' << std::get<1>(s) << '\n'; // use std::get<n> to get the nth element of the tuple
 
	return 0;
}

这与前面的例子相同。

还可以使用std::tie将元组分解预定义变量，如下所示:

#include <tuple>
#include <iostream>
 
std::tuple<int, double> returnTuple() // return a tuple that contains an int and a double
{
	return { 5, 6.7 };
}
 
int main()
{
	int a;
	double b;
	std::tie(a, b) = returnTuple(); // put elements of tuple in variables a and b
	std::cout << a << ' ' << b << '\n'; 
 
	return 0;
}

# conclusion

​ 大多数情况下，按值返回就足以满足您的需求。这也是将信息返回给调用者的最灵活和最安全的方式。然而，通过引用或地址返回也可能是有用的，特别是在处理动态分配的类或结构时。当使用返回引用或返回值时，请确保您返回的不是函数返回时超出作用域的引用或变量的地址。
