# Class template specialization

​ 上一章节谈到了函数模板的特化，我们知道是可以使用特化函数来为不同数据类型提供不同的函数。事实证明，不仅能够特化函数，也能特化一整个类。

​ 考虑一个例子，想要设计一个类，能够存储8个object，我们可以简单地这么做：

template <class T>
class Storage8
{
private:
    T m_array[8];
 
public:
    void set(int index, const T &value)
    {
        m_array[index] = value;
    }
 
    const T& get(int index) const
    {
        return m_array[index];
    }
};

因为这个类是模板化的，所以它适用于任何给定类型:

#include <iostream>
 
int main()
{
    // Define a Storage8 for integers
    Storage8<int> intStorage;
 
    for (int count{ 0 }; count < 8; ++count)
        intStorage.set(count, count);
 
    for (int count{ 0 }; count < 8; ++count)
        std::cout << intStorage.get(count) << '\n';
 
    // Define a Storage8 for bool
    Storage8<bool> boolStorage;
    for (int count{ 0 }; count < 8; ++count)
        boolStorage.set(count, count & 3);
 
	std::cout << std::boolalpha;
 
    for (int count{ 0 }; count<8; ++count)
    {
        std::cout << boolStorage.get(count) << '\n';
    }
 
    return 0;
}

​ 虽然这个类是完全可用的，但事实证明，Storage8<bool>的实现比它需要的效率要低得多。因为所有的变量都必须有一个地址，而且CPU不能处理小于一个字节的数据，所以所有的变量都必须至少有一个字节的大小。因此，bool类型的变量最终会使用整个字节，尽管技术上它只需要一个位来存储其true或false值!因此，bool是1位有用的信息和7位浪费的空间。我们的Storage8<bool>类包含8个bool，它包含1个字节的有用信息和7个字节的浪费空间。

​ 事实证明，使用一些基本的位逻辑，可以将所有8个bool压缩为一个字节，从而完全消除了浪费的空间。然而，为了做到这一点，当与bool类型一起使用时，我们需要修改该类，将包含8个bool的数组替换为一个单个字节大小的变量。虽然我们可以创建一个全新的类来实现此目的，但这有一个主要缺点:我们必须给它一个不同的名称。然后程序员必须记住，Storage8<T>意味着非bool类型，而Storage8Bool(或任何我们命名的新类)意味着bool类型。我们希望避免不必要的复杂性。幸运的是，c++为我们提供了一个更好的方法:类模板特化。

# Class template specialization

​ 类模板特化允许我们特化有特定数据类型(或者一些数据类型，如果有多个模板形参)的模板类。在本例中，我们将使用类模板特化来编写Storage8<bool>的自定义版本，该版本将优先于泛型Storage8<T>类。这类似于专门化函数优先于泛型模板函数。

​ 类模板特化被视为完全独立的类，即使它们的分配方式与模板类相同。这意味着我们可以改变专门化类的一切，包括它的实现方式，甚至是它公开的函数，就像它是一个独立的类一样。这是我们的特化类:

template <> // the following is a template class with no templated parameters
class Storage8<bool> // we're specializing Storage8 for bool
{
// What follows is just standard class implementation details
private:
    unsigned char m_data{};
 
public:
    void set(int index, bool value)
    {
        // Figure out which bit we're setting/unsetting
        // This will put a 1 in the bit we're interested in turning on/off
        auto mask{ 1 << index };
 
        if (value)  // If we're setting a bit
            m_data |= mask;  // Use bitwise-or to turn that bit on
        else  // if we're turning a bit off
            m_data &= ~mask;  // bitwise-and the inverse mask to turn that bit off
	}
	
    bool get(int index)
    {
        // Figure out which bit we're getting
        auto mask{ 1 << index };
        // bitwise-and to get the value of the bit we're interested in
        // Then implicit cast to boolean
        return (m_data & mask);
    }
};

​ 现在，当我们声明一个类型为Storage8<T>的类(其中T不是bool类型)时，我们将从泛型模板存储age8<T>类获得一个模板版本。当我们声明一个类型为Storage8<bool>的类时，我们将得到刚刚创建的专门化版本。请注意，我们保持了两个类的公开接口相同——尽管c++允许我们随意添加、删除或更改Storage8<bool>的函数，但保持一致的接口意味着程序员可以以完全相同的方式使用这两个类。

​ 我们可以使用与前面完全相同的例子来显示Storage8和Storage8被实例化:

int main()
{
    // Define a Storage8 for integers (instantiates Storage8<T>, where T = int)
    Storage8<int> intStorage;
 
    for (int count{ 0 }; count < 8; ++count)
    {
        intStorage.set(count, count);
	}
 
    for (int count{ 0 }; count<8; ++count)
    {
        std::cout << intStorage.get(count) << '\n';
    }
 
    // Define a Storage8 for bool  (instantiates Storage8<bool> specialization)
    Storage8<bool> boolStorage;
    
    for (int count{ 0 }; count < 8; ++count)
    {
        boolStorage.set(count, count & 3);
    }
 
	std::cout << std::boolalpha;
 
    for (int count{ 0 }; count < 8; ++count)
    {
        std::cout << boolStorage.get(count) << '\n';
    }
 
    return 0;
}

​ 值得再次注意的是，保持模板类和所有特化之间的公共接口相同通常是一个好主意，因为这使它们更易于使用——但是，这并不是严格必要的。
