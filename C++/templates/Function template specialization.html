# Function template specialization

​ 当使用给定的类型去实例化一个模板函数时，编译器会复制一个模板函数，并且使用实际的类型去取代模板类型参数。这意味着一个特殊的函数会对每一种类型有相同的实现细节。大部分情况下，这就是希望得到的。但是，某些情况下，对于特定类型的实现一个不同的模板函数也是有用的。模板特殊化就是实现这种方式的一种方法。

​ 让我们来看一个简单的模板类：

template <class T>
class Storage
{
private:
    T m_value;
public:
    Storage(T value)
    {
         m_value = value;
    }
 
    ~Storage()
    {
    }
 
    void print()
    {
        std::cout << m_value << '\n';
    }
};

上面这个模板能够工作在多种类型上

int main()
{
    // Define some storage units
    Storage<int> nValue(5);
    Storage<double> dValue(6.7);
 
    // Print out some values
    nValue.print();
    dValue.print();
}

​ 但是现在，我们希望仅仅是double类型的输出有些许不同。为了做到这个，我们就能使用function template specialization去创建为类型为double的print()的特殊版本。这个非常简单：简单的定义特化函数(如果函数是成员函数，可以放在外面)，使用这个特殊的版本去取代模板类。这个是我们double版本的print函数：

template <>
void Storage<double>::print(){
    std::out<<std::scientific<<m_value<<"\n";
}

​ 当编译器打算去实例化一个Storage<double>::print()，它就会看到我们早就明确地定义了这个函数，它就会使用这个我们已经定义过的，而不会再从通用的模板中创建一个新的一份。

​ template <>告诉编译器这是一个模板函数，但是它这里并没有模板参数。一些编译器允许你忽略它，但是最好包含。

# Another example

​ 让我们再看一个模板特化有用的例子。当Storage实例化char*，Storage<char*>的构造器就是以下这个：

template <>
Storage<char*>::Storage(char* value){
    m_value = value;
}
int main(void){
    string s = "xxx";
    /*...*/
    Storage<>
}

​ 换句话说，这仅仅这是一个指针的分配(pointer assignment, shallow copy)。于是，m_value最终指向的是相同的string的内存地址里。当我们在main删除这个字符串时，我们最终删除了m_value指向的地方，最后当我们尝试打印时，就会得到错误的输出。

​ 幸运地是，我们可以使用模板特化来修正这个问题。并不是一个指针的复制，我们希望我们的构造器能够复制输入的字符串。所以，让我们写一个类型为char*的特例构造器吧：

template<>
Storage<char*>::Storage(char* value){
    // Figure out how long the string in value is
    int length = 0;
    while(value[length] != '\0'){
        ++length;
    }
    ++length; //  +1 to account for null terminator
    // Allocate memory to hold the value string
    m_value = new char[length];
    // Copy the actual value string into the m_value memory we just allocated
    for (int count=0; count < length; ++count)
        m_value[count] = value[count];
}

现在，当我们分配类型为Storage <char *>的变量时，将使用该构造函数代替默认构造函数。结果，m_value将收到其自己的字符串副本。因此，当我们删除字符串时，m_value将不受影响。

但是，对于char类型，这个类现在有内存泄漏，因为当存储变量超出作用域时，m_value不会被删除。正如你可能已经猜到的，这个问题也可以通过特化Storage<char*>析构函数来解决:

template <>
Storage<char*>::~Storage()
{
    delete[] m_value;
}

现在，当类型为Storage<char*>的变量超出作用域时，在特化构造函数中分配的内存将在特化析构函数中删除。

虽然上面的示例都使用的是成员函数，但您也可以用同样的方式特化非成员模板函数。
