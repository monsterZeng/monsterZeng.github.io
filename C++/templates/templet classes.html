# Template classes

​ 函数模板让我们能够建立一个不同参数也能使用的通用函数。但是，这仅仅解决了一部分问题，我们还需要利用模板解决其它问题。

# Templates and container classes

​ 以下实现一个简单的int类型的array容器。

#ifndef INTARRAY_H
#define INTARRAY_H
 
#include <cassert>
 
class IntArray
{
private:
    int m_length{};
    int *m_data{};
 
public:
    IntArray(int length)
    {
        assert(length > 0);
        m_data = new int[length]{};
        m_length = length;
    }
 
    // We don't want to allow copies of IntArray to be created.
    IntArray(const IntArray&) = delete;
    IntArray& operator=(const IntArray&) = delete;
 
    ~IntArray()
    {
        delete[] m_data;
    }
    
    void Erase()
    {
        delete[] m_data;
        // We need to make sure we set m_data to 0 here, otherwise it will
        // be left pointing at deallocated memory!
        m_data = nullptr;
        m_length = 0;
    }
 
    int& operator[](int index)
    {
        assert(index >= 0 && index < m_length);
        return m_data[index];
    }
 
    int getLength() const { return m_length; }
};
 
#endif

​ 一旦我们想要double型的array时，我们就需要重新实现这种类型的容器，但是功能是一致的，仅仅是改变了类型了。

#ifndef DOUBLEARRAY_H
#define DOUBLEARRAY_H
 
#include <cassert>
 
class DoubleArray
{
private:
    int m_length{};
    double *m_data{};
 
public:
 
    DoubleArray(int length)
    {
        assert(length > 0);
        m_data = new double[length]{};
        m_length = length;
    }
 
    DoubleArray(const DoubleArray&) = delete;
    DoubleArray& operator=(const DoubleArray&) = delete;
 
    ~DoubleArray()
    {
        delete[] m_data;
    }
 
    void Erase()
    {
        delete[] m_data;
        // We need to make sure we set m_data to 0 here, otherwise it will
        // be left pointing at deallocated memory!
        m_data = nullptr;
        m_length = 0;
    }
 
    double& operator[](int index)
    {
        assert(index >= 0 && index < m_length);
        return m_data[index];
    }
 
    int getLength() const { return m_length; }
};
 
#endif

​ 可以看到，int类型和double类型的数组几乎是相同的。仅仅是包含的数据类型不同。这也是一个模板能够应用到的地方，创建一个类而不用实现明确其类型。

​ 跟创建函数模板一样，首先我们要声明一个模板的类型。

#ifndef ARRAY_H
#define ARRAY_H
 
#include <cassert>
 
template <class T>
class Array
{
private:
    int m_length{};
    T *m_data{};
 
public:
 
    Array(int length)
    {
        assert(length > 0);
        m_data = new T[length]{};
        m_length = length;
    }
 
    Array(const Array&) = delete;
    Array& operator=(const Array&) = delete;
 
    ~Array()
    {
        delete[] m_data;
    }
 
    void Erase()
    {
        delete[] m_data;
        // We need to make sure we set m_data to 0 here, otherwise it will
        // be left pointing at deallocated memory!
        m_data = nullptr;
        m_length = 0;
    }
 
    T& operator[](int index)
    {
        assert(index >= 0 && index < m_length);
        return m_data[index];
    }
 
    // templated getLength() function defined below,声明的是一种只读函数
    int getLength() const; 
};
 
// member functions defined outside the class need their own template declaration
template <class T>
int Array<T>::getLength() const // note class name is Array<T>, not Array
{
  return m_length;
}
 
#endif

​ 模板类被实例化跟模板函数被初始化一样，根据需求，编译器会用实际类型取代模板参数，然后编译器拷贝一份。同样的，如果你没有使用模板类，那么编译器是不会编译它的。

​ 模板类是实现容器类的理想选择，因为非常需要使容器能够处理多种数据类型，并且模板允许您这样做而无需重复代码。尽管语法可能很奇怪，而且错误消息可能很含糊，但是模板类确实是C ++最好和最有用的功能之一。

# Template classes in the standard library

​ 所以，当我们使用std::vector时，必须要明确元素的类型。例如使用int类型的容器：std::vector<int> a。

# Splitting up template classes

​ 模板既不是类也不是函数，它仅仅时被用来创建类和函数的一种范式。因此，它不能以与正常函数或类完全相同的方式工作。在大多数情况下，这不是什么大问题。但是，有一个领域通常会给开发人员造成问题。

​ 对于非模板类，常见的过程是将类的定义放在头文件中，将成员函数的定义放在命名类似的代码文件中。通过这种方式，类的源代码被编译为一个单独的项目文件。然而，对于模板，这就不起作用了。考虑以下:

Array.h:

#ifndef ARRAY_H
#define ARRAY_H
 
#include <cassert>
 
template <class T>
class Array
{
private:
    int m_length{};
    T* m_data{};
 
public:
 
    Array(int length)
    {
        assert(length > 0);
        m_data = new T[length]{};
        m_length = length;
    }
 
    Array(const Array&) = delete;
    Array& operator=(const Array&) = delete;
 
    ~Array()
    {
        delete[] m_data;
    }
 
    void Erase()
    {
        delete[] m_data;
 
        m_data = nullptr;
        m_length = 0;
    }
 
    T& operator[](int index)
    {
        assert(index >= 0 && index < m_length);
        return m_data[index];
    }
 
    int getLength() const; 
};
 
#endif

Array.cpp:

#include "Array.h"
 
template <class T>
int Array<T>::getLength() const // note class name is Array<T>, not Array
{
  return m_length;
}

main.cpp

#include "Array.h"
 
int main()
{
	Array<int> intArray(12);
	Array<double> doubleArray(12);
 
	for (int count{ 0 }; count < intArray.getLength(); ++count)
	{
		intArray[count] = count;
		doubleArray[count] = count + 0.5;
	}
 
	for (int count{ intArray.getLength() - 1 }; count >= 0; --count)
		std::cout << intArray[count] << '\t' << doubleArray[count] << '\n';
 
	return 0;
}

​ 为了让编译器去使用一个模板，它必须看到模板的定义(不仅仅只是声明)和模板的类型才能去实例化一个模板。并且记住C++独立地编译每一个文件。当Array.h头文件被main.cpp包括时，模板类定义被复制到main.cpp。当编译器看到我们使用两个模板实例：Array<int>和Array<doluble>时，编译器就会实例化这两个函数，编译器就会将实例后的模板变成main.cpp的一部分。然而，当独立地编译Array.cpp，编译器就会忘记我们使用Array<int>和Array<ddouble>，以致于模板函数时不会被初始化的。因此，我们就会得到一个链接错误，因为编译器不能发现Array<int>::getLength()或者Array<double>::getLength()的定义。

/tmp/ccDge8rJ.o: In function `main':
main.cpp:(.text+0x7c): undefined reference to `Array<int>::getLength() const'
main.cpp:(.text+0x92): undefined reference to `Array<int>::getLength() const'
collect2: error: ld returned 1 exit status

​ 有很多方法可以解决这个问题:

  • 最简单的方法是将所有模板类代码简单地放在头文件中。这样的缺点是，如果在很多地方使用了这个模板类，最终会得到很多模板类的局部副本，这会增加编译和链接时间。编译器确实会移除多余的拷贝，所以可执行文件不会变得很大。是我们的首选解决方案，除非编译或链接时间开始成为一个问题。

  • 如果你觉得将成员函数的定义放在类定义里会显得很繁琐，另外一个方法就是将Array.cpp改名为Array.inl（inl代表内联inline）。并且在Array.h的底部去包括Array.inl，能够与第一种方法有相同的效果。，但是却显得整洁。

    #ifndef ARRAY_H
#define ARRAY_H
 
#include <cassert>
 
template <class T>
class Array
{
    /*...*/
    int getLength() const; 
};
# include "Array.inl" //在底部去包含inl文件
#endif

  • 另外一种将类成员函数的实现也同时在使用时包括进来，但是不推荐这样做，这不符合标准模式。
