# Partial template specialization

​ 部分模板特化并不经常使用(但在特定情况下可能有用)。在Template non-type parameters，您了解了如何使用表达式参数来参数化模板类。

​ 让我们再看一下在前面的示例之一中使用的Static Array类：

template <class T, int size> // size is the expression parameter
class StaticArray
{
private:
    // The expression parameter controls the size of the array
    T m_array[size]{};
 
public:
    T* getArray() { return m_array; }
	
    T& operator[](int index)
    {
        return m_array[index];
    }
};

​ 该类具有两个模板参数，一个类型参数和一个表达式参数。 现在，假设我们要编写一个函数以打印出整个数组。尽管我们可以将其实现为成员函数，但我们将其作为非成员函数来实现，因为它将使后续示例更容易理解。

​ 使用模板，我们可能会这样写：

template <typename T, int size>
void print(StaticArray<T, size> &array)
{
    for (int count{ 0 }; count < size; ++count)
        std::cout << array[count] << ' ';
}

​ 这将使我们能够执行以下操作：

#include <iostream>
#include <cstring>
 
template <class T, int size> // size is the expression parameter
class StaticArray
{
private:
	// The expression parameter controls the size of the array
	T m_array[size]{};
 
public:
	T* getArray() { return m_array; }
 
	T& operator[](int index)
	{
		return m_array[index];
	}
};
 
template <typename T, int size>
void print(StaticArray<T, size> &array)
{
	for (int count{ 0 }; count < size; ++count)
		std::cout << array[count] << ' ';
}
 
int main()
{
	// declare an int array
	StaticArray<int, 4> int4{};
	int4[0] = 0;
	int4[1] = 1;
	int4[2] = 2;
	int4[3] = 3;
 
	// Print the arraypp
	print(int4);
 
	return 0;
}

尽管这可行，但它具有设计缺陷。考虑以下：

int main()
{
    // declare a char array
    StaticArray<char, 14> char14{};
 
    std::strcpy(char14.getArray(), "Hello, world!");
 
    // Print the array
    print(char14);
 
    return 0;
}

该程序将编译，执行并产生以下值（或一个类似的值）：

H e l l o ,   w o r l d !  //中间有空格

​ 对于非字符类型，在每个数组元素之间放置一个空格是有意义的，因此它们不能一起运行。但是，对于char类型，将所有一起运行的内容打印为C样式的字符串更有意义，而我们的print（）函数则不这样做。

​ 那么我们该如何解决呢？

模板特化可以解救吗

​ 可能首先想到使用模板特化。完全模板特化的问题在于必须明确定义所有模板参数。

#include <iostream>
#include <cstring>
 
template <class T, int size> // size is the expression parameter
class StaticArray
{
private:
	// The expression parameter controls the size of the array
	T m_array[size]{};
 
public:
	T* getArray() { return m_array; }
 
	T& operator[](int index)
	{
		return m_array[index];
	}
};
 
template <typename T, int size>
void print(StaticArray<T, size> &array)
{
	for (int count{ 0 }; count < size; ++count)
		std::cout << array[count] << ' ';
}
 
// Override print() for fully specialized StaticArray<char, 14>
template <>
void print(StaticArray<char, 14> &array)
{
	for (int count{ 0 }; count < 14; ++count)
		std::cout << array[count];
}
 
int main()
{
    // declare a char array
    StaticArray<char, 14> char14{};
 
    std::strcpy(char14.getArray(), "Hello, world!");
 
    // Print the array
    print(char14);
 
    return 0;
}

如您所见，我们现在为完全特化的StaticArray <char，14>提供了重载的打印功能。确实，此打印：

Hello, world!

尽管这解决了确保可以使用StaticArray <char，14>调用print()的问题，但它带来了另一个问题：使用完整模板专用化意味着我们必须显式定义此函数将接受的数组的长度！考虑以下示例：

int main()
{
    // declare a char array
    StaticArray<char, 12> char12{};
 
    std::strcpy(char12.getArray(), "Hello, mom!");
 
    // Print the array
    print(char12);

​ 用char12调用print()将调用带有staticArray <T，size>的print()版本，因为char12的类型为StaticArray <char，12>，并且只有在传递StaticArray<char，14>时才会调用我们的重载后的print() 。 尽管我们可以复制处理staticArray <char，12>的print（）的副本，但是当我们要调用数组大小为5或22的print（）时会发生什么呢？我们必须针对每个不同的数组大小复制该函数。那是多余的。 显然，完整的模板专业化在这里过于严格。我们正在寻找的解决方案是部分模板专业化。

Partial template specialization

​ 部分模板特化使我们能够特化一些类（但不是单个函数！），其中已经明确定义了一些（但不是全部）模板参数。对于上述困难，理想的解决方案是让我们的重载打印函数与char类型的StaticArray一起使用，但将长度表达式参数设置为模板，以便可以根据需要进行更改。部分模板特化使我们能够做到这一点！

// overload of print() function for partially specialized StaticArray<char, size>
template <int size> // size is still a templated expression parameter
void print(StaticArray<char, size> &array) // we're explicitly defining type char here
{
	for (int count{ 0 }; count < size; ++count)
		std::cout << array[count];
}

​ 如您所见，我们已明确声明该函数仅适用于char类型的StaticArray，但是size仍然是模板化的表达式参数，因此它将适用于任何大小的char数组。这里的所有都是它的！

​ 这是使用此功能的完整程序：

#include <iostream>
#include <cstring>
 
template <class T, int size> // size is the expression parameter
class StaticArray
{
private:
	// The expression parameter controls the size of the array
	T m_array[size]{};
 
public:
	T* getArray() { return m_array; }
 
	T& operator[](int index)
	{
		return m_array[index];
	}
};
 
template <typename T, int size>
void print(StaticArray<T, size> &array)
{
	for (int count{ 0 }; count < size; ++count)
		std::cout << array[count] << ' ';
}
 
// overload of print() function for partially specialized StaticArray<char, size>
template <int size>
void print(StaticArray<char, size> &array)
{
	for (int count{ 0 }; count < size; ++count)
		std::cout << array[count];
}
 
int main()
{
	// Declare an char array of size 14
	StaticArray<char, 14> char14{};
 
	// Print the array
	print(char14);
 
	// Now declare an char array of size 12
	StaticArray<char, 12> char12{};
 
	std::strcpy(char12.getArray(), "Hello, mom!");
 
	// Print the array
	print(char12);
 
	return 0;

​ 请注意，从C ++ 14开始，部分模板特化只能与类一起使用，而不能与模板函数一起使用（功能必须完全特化）。我们的void print(StaticArray <char，size>＆array)示例之所以有效，是因为print函数不是部分特化的（它只是使用部分特化的类参数的重载函数）。

Partial template specialization for member functions

​ 函数部分特化的局限性在处理成员功能时可能会带来一些挑战。例如，如果我们这样定义StaticArray怎么办？

template <class T, int size> // size is the expression parameter
class StaticArray
{
private:
    // The expression parameter controls the size of the array
    T m_array[size]{};
 
public:
    T* getArray() { return m_array; }
	
    T& operator[](int index)
    {
        return m_array[index];
    }
 
    void print()
    {
        for (int i{ 0 }; i < size; ++i)
            std::cout << m_array[i] << ' ';
        std::cout << '\n';
    }
};

现在，print()是StaticArray <T，int>类的成员函数。那么，当我们想要部分专门化print()使其工作方式不同时会发生什么呢？您可以尝试以下方法：

// Doesn't work
template <int size>
void StaticArray<double, size>::print()
{
	for (int i{ 0 }; i < size; ++i)
		std::cout << std::scientific << m_array[i] << ' ';
	std::cout << '\n';
}

不幸的是，这是行不通的，因为我们正在尝试部分特化函数，这是不允许的。 那么我们如何解决呢？一种明显的方法是对整个类进行部分专业化处理：

#include<iostream>
 
template <class T, int size> // size is the expression parameter
class StaticArray
{
private:
	// The expression parameter controls the size of the array
	T m_array[size]{};
 
public:
	T* getArray() { return m_array; }
 
	T& operator[](int index)
	{
		return m_array[index];
	}
	void print()
	{
		for (int i{ 0 }; i < size; ++i)
			std::cout << m_array[i] << ' ';
		std::cout << "\n";
	}
};
 
template <int size> // size is the expression parameter
class StaticArray<double, size>
{
private:
	// The expression parameter controls the size of the array
	double m_array[size]{};
 
public:
	double* getArray() { return m_array; }
 
	double& operator[](int index)
	{
		return m_array[index];
	}
	void print()
	{
		for (int i{ 0 }; i < size; ++i)
			std::cout << std::scientific << m_array[i] << ' ';
		std::cout << '\n';
	}
};
 
int main()
{
	// declare an integer array with room for 6 integers
	StaticArray<int, 6> intArray{};
 
	// Fill it up in order, then print it
	for (int count{ 0 }; count < 6; ++count)
		intArray[count] = count;
 
	intArray.print();
 
	// declare a double buffer with room for 4 doubles
	StaticArray<double, 4> doubleArray{};
 
	for (int count{ 0 }; count < 4; ++count)
		doubleArray[count] = (4.0 + 0.1 * count);
 
	doubleArray.print();
 
	return 0;
}

虽然它工作，这不是一个好的解决方案，因为我们必须复制大量代码从StaticArray<T，size>到StaticArray<double，size>。要是有办在StaticArray<T, size> 重用StaticArray<double, size>中的代码就好了。听起来像是继承的工作!你可能开始尝试这样写代码:

template <int size> // size is the expression parameter
class StaticArray<double, size>: public StaticArray< // Then what?

我们如何引用StaticArray?我们不能。幸运的是，有一个解决方案，使用一个公共基类:

#include<iostream>
 
template <class T, int size> // size is the expression parameter
class StaticArray_Base
{
protected:
	// The expression parameter controls the size of the array
	T m_array[size]{};
 
public:
	T* getArray() { return m_array; }
 
	T& operator[](int index)
	{
		return m_array[index];
	}
 
	void print()
	{
		for (int i{ 0 }; i < size; ++i)
			std::cout << m_array[i];
		std::cout << '\n';
	}
 
	virtual ~StaticArray_Base() = default;
};
 
template <class T, int size> // size is the expression parameter
class StaticArray: public StaticArray_Base<T, size>
{
public:
};
 
template <int size> // size is the expression parameter
class StaticArray<double, size>: public StaticArray_Base<double, size>
{
public:
 
	void print()
	{
		for (int i{ 0 }; i < size; ++i)
			std::cout << std::scientific << this->m_array[i] << ' ';
// note: The this-> prefix in the above line is needed.
// See https://stackoverflow.com/a/6592617 or https://isocpp.org/wiki/faq/templates#nondependent-name-lookup-members for more info on why.
		std::cout << '\n';
	}
};
 
int main()
{
	// declare an integer array with room for 6 integers
	StaticArray<int, 6> intArray{};
 
	// Fill it up in order, then print it
	for (int count{ 0 }; count < 6; ++count)
		intArray[count] = count;
 
	intArray.print();
 
	// declare a double buffer with room for 4 doubles
	StaticArray<double, 4> doubleArray{};
 
	for (int count{ 0 }; count < 4; ++count)
		doubleArray[count] = (4.0 + 0.1 * count);
 
	doubleArray.print();
 
	return 0;
}
